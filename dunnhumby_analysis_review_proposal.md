# Dunnhumby 데이터 분석 검증 및 고도화 제안서

제공해 주신 이전 분석 결과물(GitHub)의 로직과 보고서를 검토한 결과, 다음과 같은 개선 사항과 고도화 방안을 제안합니다. 본 제안서는 시계열 분석과 MBA 분석을 각각 독립적인 파트로 나누어 구성하였습니다.

---

## PART 1. Dunnhumby 시계열 수요 예측 분석 검증 (#ARIMA/Prophet)

### 1.1 사용자 요청 반영 사항: 품목명 명확화
*   **현황**: `COMMODITY_DESC`가 'COUPON/MISC ITEMS'로 표시되어 실질적인 상품 식별이 어려움.
*   **제안**: 해당 명칭 발견 시 `SUB_COMMODITY_DESC` 또는 `DEPARTMENT` 값을 함께 명시하도록 로직 수정.
    *   예: `COUPON/MISC ITEMS` -> `COUPON/MISC ITEMS (GASOLINE-REG UNLEADED)`
    *   이를 통해 주유 서비스 등 구체적인 매출 항목을 파악할 수 있습니다.

### 1.2 기능 및 로직 개선 제안
*   **모델 학습의 안정성 강화 (Data Integrity Check)**: 시계열 데이터의 길이가 백테스트 기간(4주)보다 짧거나 유사한 경우 모델 학습 시 오류가 발생할 수 있습니다. 각 상품별로 최소 데이터 포인트(예: 최소 12주 이상)를 체크하는 로직을 추가하여 예측의 신뢰성을 보장합니다.
*   **SARIMA 계절성 파라미터 최적화**: 현재 `m=1`(비계절성)로 설정되어 있어 주간 데이터의 장기적인 연간 패턴(52주 주기)을 놓칠 수 있습니다. 데이터가 충분한 상품에 대해서는 `m=52`를 적용하거나, Prophet의 계절성 컴포넌트 강도를 조절하여 정확도를 높입니다.
*   **시각화 고도화 (Visualization Enhancements)**: Prophet뿐만 아니라 SARIMA 예측치에도 신뢰 구간(Confidence Interval)을 추가하여 예측의 불확실성을 시각적으로 표현합니다.

### 1.3 리포팅 고도화 제안
*   **예측 신뢰도 점수 (Confidence Score) 도입**: 백테스팅 결과(MAE, RMSE)를 기반으로 각 상품별 예측 결과에 '상/중/하' 형태의 신뢰도 지표를 추가합니다. (예: MAE/평균 < 15% 시 "높은 신뢰도")
*   **모델 선택 자동화 (Best Model Selection)**: Prophet과 SARIMA 중 백테스팅 오차가 더 적은 모델을 해당 상품의 '최적 모델'로 선정하고 요약 보고서에 우선 표기하는 로직을 도입합니다.

## PART 2. MBA 연관규칙 분석 검증 (#Cross-Selling)

### 2.1 데이터 안정성 검증 (Stability Analysis)의 고도화
*   **현황**: 현재 93.01%의 높은 안정성을 보이고 있으나, 이는 단순히 규칙의 존재 여부만 체크함.
*   **제안**: 규칙의 `Lift`나 `Confidence` 값의 변동 계수(CV)를 함께 측정하여, 단순히 '자주 등장하는' 조합이 아닌 '효과가 일관된' 조합을 선별하는 로직을 추가합니다.

### 2.2 고객 세그먼트별 차별화된 MBA (Segmented MBA)
*   **현황**: 전체 고객 샘플을 대상으로 통합 분석을 수행함.
*   **제안**: RFM 분석 결과와 연계하여 **VIP 고객용 번들**과 **이탈 위기 고객용 번들**을 분리 도출합니다. VIP 고객은 고부가가치 상품 조합, 이탈 위기 고객은 필수 생필품 조합 등으로 마케팅 전략을 다각화할 수 있습니다.

### 2.3 계절성 및 시계열 기반 연관 분석 (Temporal MBA)
*   **제안**: 분기별 또는 시즌별로 연관 규칙의 변화를 추적합니다. '여름철 파워 번들'과 '겨울철 파워 번들'이 다를 수 있음을 데이터로 증명하여 계절별 진열 전략을 제안합니다.

## PART 3. NBA 가구별 추천 분석 검증 (#Recommendation)

### 3.1 하이브리드 추천 엔진으로의 고도화
*   **현황**: 사용자 기반 협업 필터링(User-based CF)을 사용함. 이는 구매 이력이 적은 '신규 가구(Cold Start)'에 대해 추천이 어려운 한계가 있음.
*   **제안**: 가구의 인구통계학적 특성(연령, 소득)을 활용한 콘텐츠 기반 필터링(CBF)을 결합한 **하이브리드 엔진** 도입을 제안합니다.

### 3.2 추천의 질적 지표 도입 (Beyond Accuracy)
*   **제안**: 단순히 '자주 팔리는' 카테고리가 아닌, **다양성(Diversity)**과 **의외성(Serendipity)** 지표를 측정합니다. 고객이 이전에 구매하지 않았으나 선호할 가능성이 높은 새로운 카테고리를 추천하여 교차 판매 범위를 넓힙니다.

### 3.3 개인화된 추천 근거(Explainability) 명시
*   **제안**: 보고서 및 시스템에 "이 상품을 추천하는 이유"를 데이터 기반으로 노출합니다.
    *   예: "귀하와 유사한 소득 수준의 가구에서 최근 4주간 가장 많이 구매한 상품입니다."
    *   예: "이전에 구매하신 '우유'와 연관성이 높은 '시리얼' 카테고리입니다."

## PART 4. 인사이트 및 원천 데이터 가이드 (공통 적용)

사용자의 요청에 따라 모든 분석 지표, 차트, 피벗 테이블에 대해 **500자 ~ 1000자 사이의 심층 인사이트**를 추가하며, 원천 데이터 정보를 명시합니다.

### 4.1 인사이트 포함 필수 요소
*   **데이터 현상 (What)**: 시각화나 표에서 나타나는 객관적인 수치와 패턴 설명.
*   **원인 추론 (Why)**: 왜 이런 결과가 나왔는지에 대한 비즈니스적/고객 심리학적 맥락 분석.
*   **비즈니스 액션 (Action)**: 해당 데이터를 바탕으로 즉시 실행 가능한 마케팅, 진열, 가격 전략 제안.
*   **기대 효과 (Impact)**: 제안된 액션을 실행했을 때 예상되는 매출 증성 또는 효율성 개선 수치.

### 4.2 원천 데이터 정보 명시 가이드 (Data Provenance)
*   **필수 포함 사항**: 모든 요약 및 상세 보고서의 서론 또는 개요 섹션에 분석에 사용된 **원천 데이터셋 정보**를 명확히 기재합니다.
*   **항목**:
    *   사용된 구체적인 파일명 (예: `master_transaction_table.parquet`, `product.csv` 등)
    *   데이터의 기준 기간 및 필터링 조건
*   **목적**: 보고서의 객관성과 신뢰성을 확보하고, 추후 데이터 업데이트 시 비교 기준을 명확히 하기 위함입니다.

### 4.3 한국어 표기 규정 (Korean Localization)
*   **대상**: 모든 리포트의 **제목(Title), 지표명(Metric Name), 차트 축 레이블(Axis Label), 데이터 테이블 헤더** 등.
*   **규정**: 분석 결과물 내의 모든 텍스트는 원칙적으로 **한국어**로 표기합니다.
    *   예: `TotalAmount` -> `총 매출액`
    *   예: `Future Demand Forecast` -> `향후 수요 예측 결과`
*   **목적**: 최종 보고서의 가독성을 높이고 비즈니스 의사 결정권자가 직관적으로 내용을 이해할 수 있도록 하기 위함입니다.
